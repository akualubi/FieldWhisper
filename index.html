Vision

让历史可被检验

历史问答的难点不是“生成”，而是“不胡说”——LLM 随口能提到《新唐书》、《资治通鉴》，但引用的句子可能压根不存在。史实错误、幻觉与“充满班味”的语言，是传统生成式历史问答最难跨越的三个坎。

ChronoMirror 以 Qwen2.5-VL-7B 为生成基座、GME (gme-Qwen2-VL-7B-Instruct) 为多模态嵌入引擎，通过 DashVector 向量库进行史料检索——6 类专项 Agent 各自运行 Agentic 多轮检索循环，LLM 主动判断"是否已有足够上下文"再决定是否继续检索，生成 可追溯、可被评测脚本精确验证的 历史问答。

Agentic RAG Loop · per question

四步问答

壹

多模态编码 · Encoding

Embedder · gme-Qwen2-VL-7B

文本走 text 向量（3584-d）；VQA 题额外编码图像，走 fused 多模态向量。向量提交 DashVector 检索。

贰

多轮检索 · Multi-Round Search

Retriever · DashVector · max 4 rounds

最多 4 轮检索：每轮 LLM 生成新 query，去重后查 DashVector top-k；VQA 双字段（text + fused）检索再合并去重。

叁

LLM 相关性过滤 · Filter

Qwen2.5-VL · filter + decide

LLM 阅读当轮召回结果，输出 JSON：relevant_ids 保留相关史料；enough=true 则不再追加检索轮次。

肆

答案生成 · Generation

Generator · Qwen2.5-VL-7B

累积的相关史料 + 原题（含图像）→ Qwen2.5-VL 在专项 prompt 下生成答案；输出经 normalize_answer 归一化写入 predictions.jsonl。

Research Context · §2 of Proposal

研究语境

三栏对照看清 ChronoMirror 在文献与产业景观中的位置：现状有什么 / 缺什么 / 我们做什么。

State of the Field

现状

传统 RAG 系统在通用问答任务上已趋成熟；LLM 领域亦有 Self-RAG / GraphRAG / HyDE 等增强方案。但面向专业领域（如历史）的多模态 RAG 仍在起步。

  • Qwen2.5-VL / GPT-4o 取得 SOTA 视觉问答能力
  • BGE-M3 / E5 为中文事实问答带来大幅检索提升
  • Self-RAG 证明「检索是否调用」可学习
  • history-domain QA 大多停在纯文本、未打通图像路径
The Gap

差距

跨越「图文混合输入」与「多轮检索」的历史问答系统仍是空白；现有方案处理「看图说史」基本停留在「点 OCR 加一句描述」的敷衍级调用。

  • 没有针对「题型」动态选型检索与生成策略
  • 没有可点击、可验证的引用交互
  • 多轮追问场景下跨轮检索几乎未被优化
  • 评测维度缺「引用一致性 citation faithfulness」
Our Approach

ChronoMirror 的方法

四次提交逻辑递进：Sub 1 建立 File-Bridge IPC；Sub 2 分析离线 KB、实现在线混合检索+RRF+Evidence Cards；Sub 3 推出 6 类 Agentic RAG Agent（隐藏测试 0.520）；Sub 4 将离线增强与 Multi-Agent 完整融合，综合排名第一。

  • GME 双字段嵌入 3584-d：text + fused；VQA 图文联合向量
  • DashVector＋文件桥 IPC；最多 4 轮 Agentic 检索循环
  • RRF 融合 + LexicalRerank + PlanAwareRerank
  • LocalSparse 稀疏检索 + 邻块扩展（第四次提交激活）
  • ChronoClose-150 / ChronoWide-700 自建测试集防过拟合

ChronoMirror · 史鉴

Where History Answers Itself.

项目本旨

历史问答的难点不是"生成"，而是"不胡说"。LLM 随口能提及《资治通鉴》但引用句可能压根不存在。ChronoMirror 用 6 类专项 Agent 封装完整 Agentic RAG 循环——LLM 主动生成检索 query、自主判断是否继续、过滤无关史料——把可观察、可干预、可解释的多轮检索与生成过程作为可追溯答案的核心机制。

为什么这件事值得做

从技术看，它把多模态语义、领域 RAG、可控生成与可解释评测拧到一起；从应用看，它为“大模型答历史题”这个被幻觉问题困扰的场景提供一个可追溯、可评测、可交互的替代方案。

底层架构

核心组件：Embedder（gme-Qwen2-VL-7B，3584-d 向量，字段 text / fused / image）、Retriever（DashVector，file-bridge IPC；GPU 服务器无网，通过文件 RPC 桥接到 CPU 服务器）、Generator（Qwen2.5-VL-7B，greedy decode，max_new_tokens=256）、6 专项 Agent（ChoiceAgent / ChoiceVQAAgent / JudgeAgent / JudgeVQAAgent / ShortAnswerAgent / ShortAnswerVQAAgent）。

每道题走一次完整 Agentic 循环（最多 4 轮）：LLM 生成 query → DashVector 检索 → LLM 过滤 relevant_ids → 判断 enough → 累积上下文 → 最终答案生成。输出 predictions.jsonl + time_cost.jsonl + traces.jsonl。
